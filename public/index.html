What is Predix Edge?


Tiered HW/SW Systems

Architected to run on a range of hardware platforms to deliver tiered solutions from sensor to control server. Designed to support easy porting to new hardware platforms.

Software Platform

Container based architecture with support for multi-language applications and analytics. Fosters reuse of existing GE investment through a micro-service architecture in an industry standard manner.

DevOps Infrastructure

Implemented using DevOps paradigm to enable continuous delivery for edge services and applications. Provides better quality and reliable solutions to our business customers.

Tiered Hardware/Software Alignment

Reference Architecture

Control / Data Plane

Common Data Bus, Open-source Transport (ZeroMQ), Multiple Language Bindings (e.g. C, C++, Golang, Python, Java, Node.js)

Applications & Services

Optimizing controls (MBOC), Advanced analytics (e.g. MBOC), Rules engines/CEP, Control/Data Plane, Persistence, Logging, Service discovery/Registration

Admin & Management Services

Network Management (SDN), Configuration Management, Health Diagnostics, Resource Monitoring, Edge Management Agent, Technician Console

Gateway Services

MQTT, HTTP(S), DDS, CoAP, WebSocket/SSH, TLS

Security

Certificate Management, User Management, Firewall

Device Adapter

OPC-UA, MODBUS (Kepware), EGD, MTConnect (Kepware)

Continuous Integration/Development

Building Industrial IoT Applications at the EDGE

"Edge-based software-defined machines (SDMs) like blowout preventers, turbines, medical equipment, streetlights, and locomotives leverage edge-to-cloud architecture. These SDMs are evolving and leveraging the modern edge-to-cloud platform for the Industrial Internet. Now, applications can not only collect data locally and respond to changes in that data, but they can also perform meaningful localized analytics."

- Nikhil Chauhan, Director of Product Marketing for GE Digital
